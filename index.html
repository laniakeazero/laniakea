Лист 03 — фрагментисходный код

Как это устроено внутри

Фрагмент реальной FSM-логики записи клиента в Telegram-боте на aiogram 3 — выбор услуги, проверка занятости слота и запись в БД одним переходом состояния.

booking_handlers.py 
1@router.callback_query(BookingFSM.choosing_slot)2async def select_slot(callback: CallbackQuery, state: FSMContext):3    slot_id = int(callback.data.split("_")[-1])4    # проверяем, что слот ещё свободен на момент нажатия5    async with db.session() as session:6        slot = await session.get(Slot, slot_id, with_for_update=True)78        if slot is None or slot.is_booked:9            await callback.answer("Слот уже заняли, выберите другой", show_alert=True)10            return await show_available_slots(callback.message, state)1112        slot.is_booked = True13        await state.update_data(slot_id=slot.id, master_id=slot.master_id)14        await session.commit()1516    await state.set_state(BookingFSM.confirming)17    await callback.message.edit_text(18        format_confirmation(slot), reply_markup=confirm_kb()19    )

Почему это важно: запись слота идёт через блокировку строки в БД (with_for_update) — двое клиентов физически не смогут занять один и тот же слот одновременно, даже если нажмут кнопку в одну секунду. Это то, что отличает рабочий продакшен-код от бота «для галочки».
